# 场景数据格式

词境的每个场景 = 成品仓 `index.html` 里 `slides` 数组的一个对象。加场景 = 往这个数组加一项。

## slide 结构

```js
{
  sceneCode: "21",              // 场景编号（两位）
  sceneName: "邮局",            // 中文名（顶栏横幅右半）
  sceneEn: "The Post Office",   // 英文名（横幅左半）
  sceneId: "postoffice",        // 唯一 id：同一 sceneId 的多个 slide = 该场景的多视角，←→ 互切
  section: "外出",              // 目录分组：房子 / 交通 / 外出
  viewName: "全景",             // 视角名（多视角才需要，如"值机"/"安检"）
  image: "images/场景21-邮局-1.jpg",   // 海报图（可选，探索模式不用）
  stage: "images/舞台21-邮局-1.jpg",   // 舞台图（无字，探索模式底图）
  hotspots: [                   // 热区：百分比坐标 + 词索引
    { w: 0, x: 42, y: 30, wd: 30, ht: 17 },   // w=words 数组下标，x/y/wd/ht=百分比框
    …
    // 面板入口： { panel: "面板id", x, y, wd, ht }
    // 场景传送门：{ portal: "目标sceneId", x, y, wd, ht }  （可选 view: "视角名"）
  ],
  words: [                      // 词库：顺序对应 hotspots 的 w 索引
    { en:"parcel", ipa:"/ˈpɑːrsl/", zh:"包裹", note:"备注", sen:"例句", senZh:"例句中文" },
    …
  ],
  panels: [                     // 可选：点开容器细看的子场景（抽屉/柜子等）
    { id:"面板1", name:"储物柜", stage:"images/舞台X-面板1.jpg", hotspots:[…], words:[…] }
  ]
}
```

## 关键约定

- **热区坐标**：`{x, y}` = 框左上角，`{wd, ht}` = 宽高，全部是相对图片的百分比（0–100）。视觉读图估框，接受偏松；`index.html` 内置编辑模式可微调
- **多视角**：同 `sceneId` 的多个 slide 用 `viewName` 区分，← → 平移切换；单视角场景左右箭头自动收起
- **↑↓ = 换场景，← → = 场景内平移视角**（空间语义，绝不跨场景窜）；slides 的排列顺序 = 空间动线
- **发音文件名**：`sounds/<slug>.m4a`（英）/ `<slug>_zh.mp3`（中）。slug = 单词小写、非字母数字转 `_`、合并连续 `_`、去首尾 `_`。规则与 `index.html` 里的 JS `slug()` 必须一致
- **地图热区**：`mapSpots` 数组按 `sceneId` 关联，场景上线自动点亮

## 加一个场景的完整流程

1. 定词库（**主场景词必须全新，不与已有词撞车**；面板/二次记忆词可复用）
2. 写生图任务书（见 `templates/`）→ 生图 → 视觉审计（见 `审计员岗位.md`）
3. 读图为每个词标热区百分比框
4. 往 `slides` 加 slide 对象（+ `mapSpots` 加地图热区）
5. 生成发音（`pipeline/发音管线/`）
6. 浏览器实测：每景热区数=词数、发音 fetch 200、横幅/箭头/传送门
7. （可选）代码海报管线出带标签海报（`pipeline/海报管线/`）
